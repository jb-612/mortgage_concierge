### Role of Callbacks in the Interaction

**ADK Callbacks execute on the backend**, within the ADK agent's runtime environment. They provide powerful hooks to observe, customize, and control the agent's behavior at various points in its lifecycle. While callbacks don't directly interact with the frontend or BFF in terms of sending data to them, they significantly influence the agent's processing and the resulting responses that the BFF will eventually relay to the website chat.

Here's how different types of callbacks can indirectly affect the website chat experience:

*   **Agent Lifecycle Callbacks (`before_agent_callback`, `after_agent_callback`):** These callbacks execute at the beginning and end of an agent's main processing. They can be used for setup, logging, or cleanup, which indirectly ensures the agent functions correctly and efficiently, leading to better responses in the chat. For example, `before_agent_callback` could check the session state and decide to skip certain agent logic based on the context, influencing what the user sees in the chat.

*   **LLM Interaction Callbacks (`before_model_callback`, `after_model_callback`):** These callbacks are crucial for controlling the interaction with the underlying Language Model.
    *   `before_model_callback`: Allows you to inspect and modify the LLM request before it's sent. This is essential for implementing **input guardrails** (e.g., blocking harmful content), adding dynamic context, or even serving cached responses. If a user enters a blocked query, this callback can return a predefined `LlmResponse`, which will be part of the events returned by the ADK API server and ultimately shown in the chat.
    *   `after_model_callback`: Allows you to inspect and modify the LLM's response after it's received. This can be used for **output filtering**, reformatting the language model's output to be more suitable for the chat interface, or logging the raw response. Any modifications here will be reflected in the final response seen by the user.

*   **Tool Execution Callbacks (`before_tool_callback`, `after_tool_callback`):** These callbacks control the execution of tools used by the agent.
    *   `before_tool_callback`: Allows you to inspect and modify the arguments passed to a tool before it's executed or even skip the tool execution entirely by returning a dictionary representing a mock tool result. This can be used for **tool argument validation** or implementing guardrails on tool usage (e.g., preventing the agent from calling certain tools with specific parameters). If a tool call is blocked, the agent's subsequent actions and the final chat response will be affected.
    *   `after_tool_callback`: Allows you to inspect and modify the result returned by a tool before it's used by the agent. This can be used for **post-processing tool outputs**, extracting relevant information, or even correcting errors in the tool's response before the agent reasons over it. Any changes here will influence the agent's next steps and the final output in the chat.

In essence, callbacks enable you to build logic into your ADK agent that reacts to and shapes the conversational flow initiated by the user through the website chat. The BFF then serves as the conduit to transmit these interactions back and forth between the frontend and the ADK backend.
